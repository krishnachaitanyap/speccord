speccord

the spec is the contract your code is checked against

A spec-driven development CLI. It works in both directions — discover the contract of an existing service, or generate code from a spec for a new one — and fails the build when the two drift apart.

Get started ▶ Live demo Deck

Extract

code → spec

Discover the as-is contract of an existing service: API, data, events, security.

Generate

spec → code

constitution → spec → plan → tasks → story → implementation, with role personas.

Enforce

keep them in sync

Lifecycle gates, a CI drift gate, and runtime conformance — deterministic, not prompts.

Why speccord

Hybrid by construction. Facts and pass/fail are deterministic — parsers, the contract-surface diff, the lifecycle state machine, the gates are all plain code. Only prose is model-written, always grounded in facts the tool already established, so the model never invents an endpoint, table, or scope.

Stack-agnostic. Discovery reads the contract artifacts that exist in any language — OpenAPI, GraphQL, Protobuf, AsyncAPI, Prisma, SQL — and enterprises can declare custom providers for proprietary frameworks.

AI-agent native. A built-in MCP server exposes speccord as tools, resources, and prompts; one command wires it into Claude Code, Cursor, and Copilot.

Install

# Node.js >= 18
git clone https://github.com/krishnachaitanyap/speccord.git
cd speccord
npm install
npm run build
npm link            # puts `speccord` on your PATH (or call ./bin/run.js)

An ANTHROPIC_API_KEY is optional — every command runs without it (prose becomes deterministic skeletons), and the deterministic guarantees never need it.

Quick start

Existing service

speccord init --service orders
speccord discover        # parse the contract surface
speccord base draft      # confirm facts → base spec
speccord conform        # ✓ code matches the baseline

New service

speccord init --service gift --greenfield
speccord base new --intent "issue & redeem gift cards"
speccord feature new --id SPEC-1 --title "Issue card"
speccord plan specs/features/SPEC-1-*.md

The forward workflow (any mode)

speccord constitution                       # project principles (P-n) [once]
speccord feature new --id SPEC-142 --title "Order cancellation"
speccord clarify   specs/features/SPEC-142-*.md   # resolve open questions
speccord plan      specs/features/SPEC-142-*.md
speccord tasks     specs/features/SPEC-142-*.md
speccord analyze   specs/features/SPEC-142-*.md   # deterministic cross-check
speccord advance   specs/features/SPEC-142-*.md --to Approved
speccord implement specs/features/SPEC-142-*.md   # prompt pack, or --execute

Enforce in CI

speccord gate --base origin/main   # contract changed but no spec updated → fail
speccord conform                   # running code drifted from baseline → fail

One knob: scale

You pick how much process with a single number; phases, roles, and capabilities derive from it and are individually overridable. speccord capabilities shows exactly what's on.

ScaleAdds
0 prototypeconformance only — no ceremony
1 small+ plan, stories, lifecycle, CI gate
2 medium (default)+ solutioning, PRD, QA review
3 large+ analysis (brief), UX
4 enterprise+ product ownership, compliance gates

AI agent integration

One command wires speccord into your coding agent as its contract layer — the agent generates, speccord verifies.

speccord agent-rules        # writes MCP config + rules for Claude Code, Cursor, Copilot
speccord mcp               # the MCP server (stdio)

The MCP server exposes tools (read / verify / transition / drive), resources (speccord://base-spec, speccord://constitution, speccord://story/{id}), and prompts (implement-next-story, review-changes, fix-drift). The agent reads the spec, writes code, runs the gates, fixes what they report, and only then advances.
